Контекст Context & Problem

Мобильное приложение онлайн-кинотеатра одной из ведущих стриминговых платформ (MAU 15млн) Mobile app of an online cinema for one of the leading streaming platforms (MAU 15m)

Когда я подключился к задаче, PIN-сценарий уже был реализован, но вызывал серьёзные проблемы: замедлял основной пользовательский путь и генерировал поток жалоб в поддержку. When I joined the project, the PIN scenario was already implemented, but it caused serious problems: it slowed down the main user journey and generated a stream of complaints to support.

Я предложил альтернативный сценарий, который требовал более сложной разработки, но давал значительно лучший опыт. На ревью с арт-директором, продактом и техлидом мы согласовали мой вариант. I proposed an alternative scenario that required more complex development but provided a significantly better experience. In a review with the art director, product manager, and tech lead we agreed on my option.

Результат: Results:

20c → 4c увеличение скорости прохождения этапа после редизайна 20s → 4s reduction in time to complete the step after redesign

1000+ часов поддержки в месяц сэкономлено 1000+ hours of support per month saved

N → 0 прекращение обращений в поддержку по теме N → 0 elimination of support requests on the topic

Что было не так с текущим решением What Was Wrong with the Current Solution

Первая версия PIN появилась в спешке, из-за требований регулятора.

Из-за этого сценарий: The first version of the PIN was rushed due to regulatory requirements.

Because of this, the scenario:

  • конфликтовал с основным пользовательским потоком conflicted with the main user flow
  • увеличивал время до просмотра контента increased time to content consumption
  • генерировал высокий поток обращений в поддержку generated a high volume of support requests
  • снижал лояльность пользователей decreased user loyalty


Проблема для бизнеса в том, что плохо спроектированный сценарий влияет на ключевой сценарий и важные для продукта метрики (удержание, возвращаемость и тп).
Business Problem is that a poorly designed scenario affects the core user journey and important product metrics (retention, return rate, etc.).

Проблема для пользователя в том, что вместо выполнения ключевого действия они сталкиваются с проблемами которые не знают как решить. User Problem is that instead of completing the core action, they encounter issues they don't know how to resolve.

Исследование и анализ Research & Discovery

Чтобы разобраться в проблемах, я составил карты пути пользователей учитывая разные роли и статусы PIN. To understand the issues, I created user journey maps considering different roles and PIN statuses.

Так же проанализировал отзывы пользователей в App Store и Google Play, и данные из службы поддержки. I also analyzed user reviews in the App Store and Google Play, as well as data from the support service.

Основные проблемные места и боли пользователей: Key problem areas and user pain points:

☹︎ нужно вводить PIN при каждом открытии карточки фильма с рейтингом 18+ ☹︎ users have to enter the PIN every time they open a movie card with an 18+ rating

☹︎ нет понятного способа отключить PIN, если он не нужен ☹︎ there is no clear way to disable the PIN if it is not needed

☹︎ пользователи не понимают зачем нужен PIN и как он работает ☹︎ users don't understand why the PIN is needed and how it works

Цели: убрать лишние шаги, сделать управление PIN понятным, объяснить ценность функции тем, кому она реально нужна Goals: eliminate unnecessary steps, make PIN management clear, explain the value of the feature to those who really need it

Решение Solution

Изначально задача стояла в реализации максимально простого и дешёвого решения, однако в процессе стало ясно, что такой подход не решает ключевые проблемы пользователей. Initially, the task was to implement the simplest and most cost-effective solution. However, it became clear during the process that this approach would not solve the core user problems.

Я предложил сразу перейти на новые компоненты, без использования legacy-элементов, чтобы обеспечить лучший пользовательский опыт и упростить дальнейшую поддержку. I proposed to switch directly to new components, without using legacy elements, to ensure a better user experience and simplify further maintenance.

Чтобы убедиться в эффективности нового варианта, я сделал прототип нового сценария и провел небольшое тестирование с пользователями. To ensure the effectiveness of the new option, I created a prototype of the new scenario and conducted a small user test.

Проработка сценариев Scenario Development

Мы сегментировали аудиторию (новые/существующие, с PIN и без), разобрали ошибки старого сценария и уточнили ключевые критичные точки. We segmented the audience (new/existing, with PIN and without), analyzed the errors of the old scenario, and clarified the key critical points.

Новый экран ввода PIN-кода New PIN Entry Screen

  • Полноэкранный формат Full-screen format
  • Лучшие практики индустрии (банковские и системные паттерны) Industry best practices (banking and system patterns)
  • Убрали всё лишнее, сделали интерфейс чище Removed all clutter, made the interface cleaner

Новый базовый сценарий New Core Scenario

  • PIN требуется только при входе в профиль PIN is only required when logging into the profile
  • Если PIN не нужен, его можно легко отключить в настройках If the PIN is not needed, it can be easily disabled in the settings
  • Добавлен онбординг для объяснения необходимости PIN Onboarding added to explain the need for a PIN

Дополнительные улучшения New Additional Features

  • Face ID / Touch ID Face ID / Touch ID
  • Автоподстановка кода из SMS при сбросе Auto-fill code from SMS during reset
  • Онбординг с объяснением, кому и когда нужен PIN Onboarding explaining who and when needs a PIN
  • Простое отключение PIN Easy PIN disabling

Итоги: Results:

После запуска обновления мы получили отличные результаты как в пользовательском опыте, так и в бизнес-метриках. Пользователи, которым был нужен PIN, стали использовать его комфортнее, а те, кому он был не нужен, смогли легко его отключить. After launching the update, we achieved excellent results in both user experience and business metrics. Users who needed a PIN could now use it more comfortably, while those who didn't need it could easily disable it.
